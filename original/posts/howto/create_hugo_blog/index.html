This is a short guide how to create a Hugo blog, using the tailwind theme, which is the one I’m currently using myself. This guide was written with unexperienced users in mind. If you follow this guide 1:1 you should get a running instance (no guarantee of course). If you are confident with the topic, you can use this guide to get inspired and modify the steps to your own preference.

Although the following steps are for Windows, the whole process is the same for Linux/MacOS. Only some of the console commands are slightly different. E.g. instead of the powershell command Start-process you can use nano or vim and instead of winget you may use your distribution’s package manager or homebrew.

Important concept to understand

The blog will be hosted on github and will be stored in two different repositories. The first repository (let’s call this repo website) is private and stores all the code and content needed to build the blog. The second repository has to be named according to your github username yourusername.github.io, where you replace yourusername with your actual github username (Please also replace yourusername in all the following code by your actual username. In my case the second repository is called jokobus.github.io). The yourusername.github.io repository contains all the public html (and other stuff) that Hugo created and is nested inside your website repository. Most of your code can be private, but the subdirectory called public (yes, that is it’s name) has to be public. Once the website is built by Hugo, we only need to publish the stuff in the public subdirectory. We publish it by pushing the content to the yourusername repository on github. All other content outside the public folder is just needed to build the content of the public folder. Finally your blog will be available on https://yourusername.github.io .

Prerequisites

On Windows, we should use the Windows Powershell and not the built in Command Prompt. If you don’t have Powershell installed, you can follow this guide . The quick way via winget is:

winget install -e --id Microsoft.PowerShell

On Linux and Mac your default Terminal should do the job.

Secondly you need to have a Github account and git installed . Create a new github account here and install git by executing this in the powershell.

winget install -e --id Git.Git

Then authenticate git to Github by one of the options mentioned in the official article .

The easiest way is to install the Github Desktop Client and authenticate there:

winget install -e --id GitHub.GitHubDesktop

Follow the installation guide and open the Github Desktop Application. Login with your Github credentials there. You will automatically be able to use your credentials with the git command.

All prerequistes met? Let’s go!

Step 1: Install Hugo

Follow the official Hugo guide . As a service I added the steps for Windows below. However, it is definitely a good idea to look at the official guide as well.

If you use winget, it is as simple as:

winget install Hugo.Hugo.Extended

Step 2: Create github repositories

Go to https://github.com and log in to your account. Create a new private repository called website and a public repository called yourusername.github.io (where you off course replace yourusername with your actual username).

Private repository

First, we clone the website repository.

  1. Move into the folder where you want to store your website and clone the repo.
cd "C:/path/to/where/you/want/to/store/your/website"
git clone https://github.com/yourusername/website.git
  1. Create a new Hugo instance.
cd website
hugo new site yourusername
cd yourusername

Optional: To see all the folders that were created, type:

ls

Do you see a config.toml or hugo.toml file? This is the configuration file, where all the settings are stored. Later, we will modify it.

Public repository

Now we clone the public yourusername.github.io repository and add it as a submodule. (Here is some unofficial, but good explanation what a submodule is. )

  1. Clone the public repository next to your private one, add a readme and push it to github
cd "C:/path/to/where/you/want/to/store/your/website"
git clone https://github.com/yourusername/yourusername.github.io.git
cd yourusername.github.io
git checkout -b main
notepad Readme.md

Add a short description like “This is the public content for my Hugo website. " to the Readme file.

git add .
git commit -m "First commit to add Readme. "
  1. Switch back into your private repository
cd ../
cd website/yourusername

and add a submodule.

git submodule add -b main https://github.com/yourusername/yourusername.github.io.git public
  1. Check that everything worked by:
cd public
ls

You should see a file called Readme.md and if you open it with

notepad Readme.md

you will find exactly the text you wrote in the Readme.md file a few lines above.

Step 3: Install theme

We use the Tailwind theme , but you can pick whichever you want. There is an extensive list of free themes on the hugo website.

cd "C:/path/to/where/you/want/to/store/your/website/website/yourusername"
git submodule add https://github.com/tomowang/hugo-theme-tailwind.git themes/tailwind

You can always update the theme by:

git submodule update --remote

Also, it is a good idea to personalize the default post header. This is the header that will be created by default each time you create a new post.

Start-process /archetypes/default.md

Replace the code in the file by the following code:

+++
title = '{{ replace .File.ContentBaseName "-" " " | title}}'
description = ""
translationKey = '{{ replace .File.ContentBaseName "-" " " | title | lower}}'
author = "Your Name"
date = {{ .Date }}
archives = ["{{ dateFormat "2006-01" .Date }}"]
tags = ["", ""]
draft = false
toc = true
+++

Step 5: Add content

To add content in markdown format you have to do the following:

  1. Create a new blog entry via
    hugo new /en/posts/yourpostname.md
    or
    hugo new /en/posts/yourpostname/index.md
    both methods work the same, but in the second variant you can also store images inside the /yourpostname/ folder.
  2. Use your favourite text editor to open the newly created markdown file and add content. You may look at some examples in /themes/tailwind/exampleSite/content/ and read this useful guide about the Markdown syntax. All the content is written in the simple Markdown format.

Multilanguage support

Tailwind supports a blog with multiple languages. That’s a really nice feature. The code for the config.toml file I provided makes your website ready for multiple languages (in this case english and german, but you can simply add another language).

The important step is, to store two .md files in the correct directories with the same translation key in their headers.

The following graphic shows the most important parts of the file-tree inside your website repository.

A few general words:

  • public In this folder all the html and other public stuff Hugo creates is stored. Only this folder needs to be public.
  • config.toml This is your configuraton file (you could also rename it to hugo.toml if you want)
  • themes this is where you clone your themes to

Now, about the multilanguage support: Ultimately we want to have two different posts with the same content, but one in english and one in german. The user can toggle on the website, which article he wants to see.

To achive this, everything is already prepared for you.

  1. Create a new article as you did in Step 5
  2. Write some english content
  3. copy index.md from /en/posts/yourpostname/index.md to /de/posts/yourpostname/index.md, open the file and translate it to german.
  4. Make sure that the headers of both index.md files have the identical value for translationKey= in their headers (e.g. both have translationKey=“guineapig”)

Step 6: Push to github

In this step, we finally get to the point where you can see the result. All you need for this is:

  1. commit the private code to the website repository
  2. Build the public content in ./public subdirectory
  3. commit the public code to the yourusername.github.io repository
  4. open https://yourusername.github.io in your browser

To do so we have to execute the following code in our terminal (on Windows use Powershell rather than cmd):

First we have to go to into our website repository

cd "C:/path/to/hugo/website/repository"
  1. commit the private code to the website repository
git add .
git commit -m "Some message"
git push origin main
  1. Build the public content in ./public subdirectory
hugo -t tailwind
cd public
  1. commit the public code to the yourusername.github.io repository
git add .
git commit -m "Some message"
git push origin main
  1. open https://yourusername.github.io in your browser
Start-Process "https://yourusername.github.io"

Step 7: Automate github interaction with script

This is way too much work! We don’t want to type half a dozen lines for just pushing our code to github. Therefore I wrote a little file that does the job for us. Disclaimer: The translation from batch to ps1 and bash was done by ChatGPT.

Note, that I built the files in a way that I can easily switch between different themes, in case I ever wish to do so. If you don’t want to use that, feel free to remove the unncessary code. Also, when the script asks for the theme name, it only takes the first character of your input. If you have two themes with the same first character, you have to modify the line that starts with for /f “delims="….

Windows Batch file (not recommended)

Create a new file called gitcommit.bat in your website repository and paste the following lines. Edit the pathtowebsite-Variable in line 3.

@echo off
title Commit Hugo Webpage to Github
set pathtowebsite = "C:/path/to/my/website"
cd pathtowebsite

set /p message="Enter Comment Message: "
set /p theme="Enter Theme Name : Tailwind (default) or Blonde: "

echo --- website Repository ---
git add -A
if "%message%"=="" (
    echo No message, committing changes without comment.
    git commit
) else (
    git commit -m "%message%"
)
git push origin
echo Total code was successfully pushed to private repository. 

echo --- public Repository ---

if "%theme%"=="" (
echo Theme Name cannot be empty, using default Tailwind
set theme = Tailwind
)

REM Convert theme to lowercase and get the first character
for /f "delims=" %%i in ('echo %theme% ^| tr "[A-Z]" "[a-z]"') do set theme=%%i

set theme=%theme:~0,1%

if "%theme%"=="b" (
    hugo -t Blonde
) else (
    hugo -t tailwind
)

cd public

git add -A
if "%message%"=="" (
    echo No message, committing changes without comment.
    git commit
) else (
    git commit -m "%message%"
)
git push origin
echo Public code successfully pushed. 

cd pathtowebsite

echo Job done, everything was pushed. 
exit

To commit your changes, type:

.\gitcommit.bat

Windows Powershell (recommended)

Create a new file called gitcommit.ps1 file in your website repository and paste the following lines. Edit the pathtowebsite-Variable in line 1.

$pathtowebsite = "C:/path/to/my/website"
Set-Location -Path $pathtowebsite

$message = Read-Host "Enter Comment Message"
$theme = Read-Host "Enter Theme Name : Tailwind (default) or Blonde"

Write-Host "--- website Repository ---"
git add -A
if ([string]::IsNullOrEmpty($message)) {
    Write-Host "No message, committing changes without comment."
    git commit
} else {
    git commit -m $message
}
git push origin
Write-Host "Total code was successfully pushed to private repository."

Write-Host "--- public Repository ---"

# Check if theme name is provided, otherwise use default
if ([string]::IsNullOrEmpty($theme)) {
    Write-Host "Theme Name cannot be empty, using default Tailwind"
    $theme = "Tailwind"
}

# Convert theme to lowercase and get the first character
$theme = $theme.ToLower()
$theme = $theme.Substring(0, 1)

if ($theme -eq "b") {
    hugo -t Blonde
} else {
    hugo -t tailwind
}

Set-Location -Path "public"

git add -A
if ([string]::IsNullOrEmpty($message)) {
    Write-Host "No message, committing changes without comment."
    git commit -m "No comment provided"
} else {
    git commit -m $message
}
git push origin
Write-Host "Public code successfully pushed."

Set-Location -Path $pathtowebsite

Write-Host "Job done, everything was pushed."
exit

To commit your changes, type:

.\gitcommit.ps1

or only

.\gitcommit

Linux bash file

Create a new file called gitcommit.sh in your website repository and paste the following lines. Edit the pathtowebsite-Variable in line 2.

#!/bin/bash
pathtowebsite="C:/path/to/my/website"
cd "$pathtowebsite"
git add -A

read -p "Enter Comment Message: " message
read -p "Enter Theme Name : Tailwind (default) or Blonde: " theme

echo "--- website Repository ---"
if [ -z "$message" ]; then
    echo "No message, committing changes without comment."
    git commit -m "No comment provided"
else
    git commit -m "$message"
fi
git push origin
echo "Total code was successfully pushed to private repository."

echo "--- public Repository ---"

if [ -z "$theme" ]; then
    echo "Theme Name cannot be empty, using default Tailwind"
    theme="Tailwind"
fi

# Convert theme to lowercase and get the first character
theme=$(echo "$theme" | tr '[:upper:]' '[:lower:]')
theme=${theme:0:1}

if [ "$theme" == "b" ]; then
    hugo -t Blonde
else
    hugo -t tailwind
fi

cd public

git add -A
if [ -z "$message" ]; then
    echo "No message, committing changes without comment."
    git commit
else
    git commit -m "$message"
fi
git push origin
echo "Public code successfully pushed."

cd "$pathtowebsite"

echo "Job done, everything was pushed."
exit 0

To commit your changes, type:

./gitcommit.sh

Step 8: Search Engine and Crawler optimization (optional)

Robots.txt

If you set ’enableRobotsTXT=true’ in hugo.toml, then Hugo automatically creates a ‘robots.txt’ file with the content:

User-agent: *

This tells web-crawlers that it is okay, to crawl your data. Wikipedia: What is a web-crawler? If you don’t want that (which I personally prefer, but that is your own mindset of course) then you can disallow all crawling by overwriting the default. To do so, create a file called ‘robots.txt’ in the ’layouts’ folder in the private ‘website’ repository. Paste

User-agent: *
Disallow: /

to disallow all crawling or

User-agent: *
Disallow: /contact
Disallow: /about
Allow: /posts

to allow only partial crawling. Modify it to your liking! For more information refer to the official Hugo Documentation

Search Engine Optimization SEO

You can generate a sitemap, which will help your website to be better found via Google. I already added the necessary section to ‘hugo.toml’ file, so the sitemap is already created. What we now have to do is to tell Google about the sitemap. Therefore we use the Google Search Console.

  1. In your web-browser:
  2. Open the Google Search Console https://search.google.com/search-console/about
  3. Log in with your Google Account if not already done so
  4. Select ‘Add Property’
  5. In the pop-up type your url https://yourdomain.github.io under ‘URL-Prefix’ (not Domain)
  6. Select HTML-Tag as your url-ownership verification method and copy the line
  7. In your private ‘website’ repository.
  8. Open the file ’themes/tailwind/layouts/partials/head.html
  9. Add the copied line at the top
  10. Save your changes and push the website to github
  11. Go back to the Google Search Console website
  12. Click on the Finish button, now Google verifies that you have control over the url. If it didn’t work right away, push the website once again to git and wait a minute or two. Then try again.
  13. Click on the sitemap entry in the sidebar, fill the field behind ‘yourusername.github.io/’ with ‘sitemap.xml’.

That’s it. Now feel free to play around with the Google Search Console.

Workflow

This is a short summary that describes all the steps you need to do each time you want to create and publish a new article.

  1. Open Powershell.

  2. Use the cd command to move into your directory, where your website is stored.

  3. Create a new content page (replace yourpostname with the actual title of your new post, but try to avoid blank spaces):

    hugo new posts/yourpostname/index.md

  4. Open index.md with your favourite text editor or IDE (e.g. Visual Studio Code) and write down your world-changing thoughts.

  5. Optional: Copy the newly created /en/posts/yourpostname/index.md to /de/posts/yourpostname/index.md and translate the text to german. Make sure to have the same values for translationKey. If you are building a multilanguage webpage, I definitely recommend that you translate every blog entry. However, it is not technically required to do so.

  6. To test that everything looks correct, execute the following in PowerShell:

    1. run the hugo server
    hugo server

    or

    hugo server --buildDrafts

    if you want all drafts to be compiled as well.

    1. open localhost:1313/ in your browser. (Every time you update content, the change will instantly be shown on localhost. )

  7. If everything is looking good on localhost and you want to make the changes public, execute our script:

    • Windows
      ./gitcommit.ps1
    • Linux
      ./gitcommit.sh

Your website should be available at yourusername.github.io.

Open your website, press F12 to go to the Network-tab, reload the page and you should see a katex.min.css and katex.min.js entry.

Dependabot alerts

After a while of running the Blog, I received Github Dependabot alerts, warning me about security issues with packages specified in the package-lock.json file. To solve those security issues you should definitely asap execute

npm update

to update those dependencies. After a new add, commit, push to github via the gitcommit script the security alerts should be solved.
