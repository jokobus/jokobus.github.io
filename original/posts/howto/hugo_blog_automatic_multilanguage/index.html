MY QUESTION

I have a Hugo blog with a lot of blog entries index.md in different subdirectories in the folder content/original/posts/postname/index.md. What i want to do now is to automatically either translate or enhance the posts with AI. First, I copy everything from content/original/posts/postname/ to content/en/posts/postname and then I want to create an api call to some chat AI that automatically translates each blog post index.md and saves this blog post in content/en/posts/postname/index.md. If same language, just enhance the text by AI. But I also have to ensure that in code blocks only the comments are translated and links to images in the content/en/posts/postname/images folder or websites stay the same etc. Help me do that.

Besides: How do I get the API calls for free?

GEMINI

Of course. You can definitely automate this process with a script. This task requires careful parsing of the file to ensure you only send the correct parts to the AI.

Here’s a comprehensive guide on how to do it, including a Python script and answers to your API question.

The Overall Workflow 🤖

Your plan is solid. The process for a script would look like this:

  1. Scan for Posts: The script will look inside content/original/posts/ for every post directory.
  2. Prepare Destination: For each post found, it will create a corresponding directory under content/en/posts/. It will also copy over non-Markdown files like images, PDFs, etc.
  3. Parse the index.md: This is the critical step. The script will read the index.md and separate it into three parts:
    • The front matter (the +++ block).
    • Regular Markdown text.
    • Code blocks (the ``` blocks).
  4. Construct an AI Prompt: It will intelligently send only the translatable text (front matter title/description and the regular Markdown) to the AI. The prompt will include strict instructions to preserve links, code syntax, and formatting.
  5. Process and Rebuild: The script will receive the translated/enhanced text from the AI and carefully rebuild the index.md file, inserting the translated text while keeping code blocks and links intact.
  6. Save the New File: The newly generated index.md will be saved in the content/en/posts/postname/ directory.

The Python Script Solution

Python is perfect for this task because it’s great at file manipulation and making API calls. Here is a well-commented script that implements the logic described above.

What you need to install first:

You’ll need Python installed, plus a couple of libraries.

pip install requests toml
  • requests: For making the API call.
  • toml: For easily parsing your +++ front matter.

The Script:

import os
import shutil
import requests
import toml
import re
from pathlib import Path

# --- CONFIGURATION ---
# Replace with your actual AI provider's details
API_KEY = "YOUR_AI_API_KEY_HERE" 
API_URL = "[https://api.openai.com/v1/chat/completions](https://api.openai.com/v1/chat/completions)" # Example for OpenAI
AI_MODEL = "gpt-4-turbo"

SOURCE_DIR = Path("content/original/posts")
TARGET_LANG = "en" # The language code for the target directory
TARGET_ROOT_DIR = Path(f"content/{TARGET_LANG}/posts")

# --- HELPER FUNCTION: THE AI CALL ---
def get_ai_translation(text, source_lang, target_lang):
    """
    Sends text to the AI for translation or enhancement.
    This is a sample implementation for OpenAI's API.
    You will need to adapt it for your chosen AI provider (e.g., Gemini, Claude).
    """
    if source_lang == target_lang:
        task = "Enhance the following text for clarity, engagement, and style. Fix any grammatical errors."
    else:
        task = f"You are a professional translator for a technical blog. Translate the following text from '{source_lang}' to '{target_lang}'."

    prompt = f"""
    {task}

    IMPORTANT RULES:
    1.  Preserve all original Markdown formatting (headings, bold, lists, etc.).
    2.  DO NOT translate or change URLs, image paths, or file links.
    3.  DO NOT translate text inside backticks (`like_this`).
    4.  The output must be only the processed text, with no extra explanations.

    Here is the text to process:
    ---
    {text}
    """

    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    }
    data = {
        "model": AI_MODEL,
        "messages": [{"role": "user", "content": prompt}],
        "temperature": 0.7
    }

    try:
        response = requests.post(API_URL, headers=headers, json=data, timeout=120)
        response.raise_for_status() # Raise an exception for bad status codes
        result = response.json()
        return result['choices'][0]['message']['content'].strip()
    except requests.exceptions.RequestException as e:
        print(f"Error calling AI API: {e}")
        return None # Return None on failure

# --- CORE SCRIPT LOGIC ---
def process_posts():
    """Main function to find and process all blog posts."""
    if not SOURCE_DIR.exists():
        print(f"Source directory not found: {SOURCE_DIR}")
        return

    print(f"Scanning for posts in {SOURCE_DIR}...")

    # Walk through all subdirectories in the source posts folder
    for post_dir in [d for d in SOURCE_DIR.iterdir() if d.is_dir()]:
        source_file = post_dir / "index.md"
        if not source_file.exists():
            continue

        print(f"\n--- Processing: {post_dir.name} ---")

        # 1. Prepare Destination Directory
        target_post_dir = TARGET_ROOT_DIR / post_dir.relative_to(SOURCE_DIR)
        target_post_dir.mkdir(parents=True, exist_ok=True)

        # 2. Copy all assets (non-md files)
        for item in post_dir.iterdir():
            if item.is_file() and item.name != "index.md":
                shutil.copy2(item, target_post_dir)
                print(f"  > Copied asset: {item.name}")

        # 3. Read and Parse the source index.md
        with open(source_file, 'r', encoding='utf-8') as f:
            content = f.read()

        # Separate front matter from markdown content
        parts = content.split('+++', 2)
        if len(parts) < 3:
            print(f"  ! Warning: Could not parse front matter for {source_file}. Skipping.")
            continue
        
        front_matter_str = parts[1]
        markdown_content = parts[2]
        
        front_matter = toml.loads(front_matter_str)
        source_lang = front_matter.get("original_language", "en") # Default to 'en' if not found

        print(f"  > Original language detected: {source_lang}")

        # --- Process Front Matter ---
        title_to_translate = front_matter.get("title", "")
        desc_to_translate = front_matter.get("description", "")
        
        if title_to_translate:
            print("  > Translating title...")
            translated_title = get_ai_translation(title_to_translate, source_lang, TARGET_LANG)
            if translated_title:
                front_matter["title"] = translated_title

        if desc_to_translate:
            print("  > Translating description...")
            translated_desc = get_ai_translation(desc_to_translate, source_lang, TARGET_LANG)
            if translated_desc:
                front_matter["description"] = translated_desc

        # --- Process Markdown Content (The Smart Way) ---
        processed_markdown = ""
        in_code_block = False
        lines = markdown_content.split('\n')

        for line in lines:
            # Toggle code block status
            if line.strip().startswith("```"):
                in_code_block = not in_code_block
                processed_markdown += line + "\n"
                continue

            # If inside a code block, only translate comments
            if in_code_block:
                # Regex to find common comments (Python/Ruby #, JS/Java //, C-style /*...*/)
                # This can be expanded for more languages
                comment_match = re.search(r'(#|//)\s*(.*)', line)
                if comment_match:
                    code_part = line[:comment_match.start(2)] # The part before the comment text
                    comment_text = comment_match.group(2)
                    print(f"  > Translating code comment: '{comment_text[:30]}...'")
                    translated_comment = get_ai_translation(comment_text, source_lang, TARGET_LANG)
                    if translated_comment:
                        processed_markdown += code_part + translated_comment + "\n"
                    else:
                        processed_markdown += line + "\n" # Add original line on failure
                else:
                    processed_markdown += line + "\n" # Add non-comment code line as-is
            else:
                # This is regular markdown text, safe to translate
                # To be more efficient, we could batch lines together, but one by one is safer
                print(f"  > Translating content line: '{line[:40]}...'")
                translated_line = get_ai_translation(line, source_lang, TARGET_LANG)
                if translated_line is not None:
                     processed_markdown += translated_line + "\n"
                else:
                     processed_markdown += line + "\n" # Add original line on failure
        
        # 4. Rebuild and Save the new file
        new_front_matter_str = toml.dumps(front_matter)
        new_content = f"+++\n{new_front_matter_str}+++\n{processed_markdown}"
        
        target_file_path = target_post_dir / "index.md"
        with open(target_file_path, 'w', encoding='utf-8') as f:
            f.write(new_content)
        
        print(f"  ✓ Successfully created: {target_file_path}")

# --- RUN THE SCRIPT ---
if __name__ == "__main__":
    process_posts()

How Do I Get API Calls for Free? 💰

This is a common and important question. Truly free, unlimited, high-quality API access is very rare because running these large models is expensive. However, you have several excellent options.

1. Free Tiers and Initial Credits (Easiest Start)

  • Google AI for Developers: Google often provides a generous free tier for the Gemini API. You can get a certain number of requests per minute for free, which is perfect for a personal project. You just need a Google account to get an API key.
  • OpenAI: When you create a new OpenAI account, you usually get some free credits (e.g., $5 worth) to use on their API. This is enough to process dozens of blog posts to test your script.
  • Anthropic (Claude): Similar to OpenAI, they often have introductory credits.

Best for: Getting started quickly and processing a small-to-medium number of posts without any setup hassle.

2. Self-Hosted Open Source Models (Most Powerful Free Option)

This is the path to truly “free” (as in no cost-per-call) API access, but it requires some technical setup and hardware.

  • What it is: You download an open-source AI model like Mistral, Llama 3, or Phi-3 and run it on your own computer.
  • How to do it: The easiest way is with a tool like Ollama. You install Ollama, then run a simple command like ollama run llama3 in your terminal. This starts a local server on your computer that acts just like the OpenAI API.
  • Adapting the script: You would change the API_URL in the Python script to your local address (e.g., http://localhost:11434/v1/chat/completions) and you often don’t need an API_KEY.
  • Hardware Requirement: You need a computer with a decent amount of RAM (16 GB+) and ideally a modern NVIDIA GPU with plenty of VRAM (8 GB+) for good performance.

Best for: Long-term, high-volume use if you’re comfortable with the command line and have the necessary hardware. This gives you complete control and privacy.

Final Tip: Start with the free credits from Google or OpenAI to get your script working perfectly. If you find yourself needing to process hundreds of posts or run the script frequently, then explore setting up a self-hosted model with Ollama.

Below is the complete, updated script. I’ve also included a major performance improvement: instead of sending every single line to the AI (which is very slow locally), this version groups all the translatable text into one big chunk and sends it in a single API call. This is vastly more efficient.

import os
import shutil
import requests
import toml
import re
from pathlib import Path

# --- CONFIGURATION for OLLAMA ---
API_KEY = "ollama" # Not needed for local Ollama, but the library might expect a value
API_URL = "http://localhost:11434/v1/chat/completions" # Ollama's local OpenAI-compatible endpoint
AI_MODEL = "llama3" # The model you downloaded with 'ollama pull llama3'

SOURCE_DIR = Path("content/original/posts")
TARGET_LANG = "en"
TARGET_ROOT_DIR = Path(f"content/{TARGET_LANG}/posts")

# --- HELPER FUNCTION: THE AI CALL (Modified for Ollama) ---
def get_ai_response(text, source_lang, target_lang):
    """
    Sends a chunk of text to the local Ollama model for translation or enhancement.
    """
    if not text.strip():
        return ""

    if source_lang == target_lang:
        task_instruction = "You are a helpful AI assistant for a technical blog. Enhance the following blog post content for clarity, engagement, and style. Fix any grammatical errors or awkward phrasing. The user will provide a series of text and code comments. Process each one."
    else:
        task_instruction = f"You are a professional translator for a technical blog. Translate the following content from '{source_lang}' to '{target_lang}'. The user will provide a series of text and code comments. Translate each one."

    # Using a structured format helps the model distinguish different pieces of text.
    prompt = f"""
    {task_instruction}

    IMPORTANT RULES:
    1.  Preserve all original Markdown formatting (headings, bold, lists, etc.).
    2.  DO NOT translate or change URLs, image paths, or file links (e.g., /images/pic.png).
    3.  DO NOT translate text inside single backticks (`like_this_variable`).
    4.  Your output MUST be ONLY the processed text, with no extra explanations or preambles.
    5.  The input text below is delimited by <<<TEXT_START>>> and <<<TEXT_END>>>. Respond with only the processed version of this text.

    <<<TEXT_START>>>
    {text}
    <<<TEXT_END>>>
    """

    headers = {
        # No 'Authorization' header is needed for local Ollama
        "Content-Type": "application/json"
    }
    data = {
        "model": AI_MODEL,
        "messages": [{"role": "user", "content": prompt}],
        "temperature": 0.5,
        "stream": False # Ensure you get the full response at once
    }

    try:
        print("  > Sending a batch of text to Ollama...")
        response = requests.post(API_URL, headers=headers, json=data, timeout=300) # Increased timeout for local models
        response.raise_for_status()
        result = response.json()
        ai_content = result['choices'][0]['message']['content'].strip()
        
        # Clean up potential model verbosity
        if ai_content.startswith("<<<TEXT_START>>>"):
            ai_content = ai_content.split("<<<TEXT_START>>>", 1)[1]
        if ai_content.endswith("<<<TEXT_END>>>"):
            ai_content = ai_content.rsplit("<<<TEXT_END>>>", 1)[0]
            
        print("  > Batch received and processed.")
        return ai_content.strip()
    except requests.exceptions.RequestException as e:
        print(f"  ! Error calling Ollama API: {e}")
        print("  ! Is the Ollama application running? You can check with 'ollama list' in your terminal.")
        return None

# --- CORE SCRIPT LOGIC (Modified for Batch Processing) ---
def process_posts():
    if not SOURCE_DIR.exists():
        print(f"Source directory not found: {SOURCE_DIR}")
        return

    print(f"Scanning for posts in {SOURCE_DIR}...")

    for post_dir in [d for d in SOURCE_DIR.iterdir() if d.is_dir()]:
        source_file = post_dir / "index.md"
        if not source_file.exists(): continue

        print(f"\n--- Processing: {post_dir.name} ---")

        target_post_dir = TARGET_ROOT_DIR / post_dir.relative_to(SOURCE_DIR)
        target_post_dir.mkdir(parents=True, exist_ok=True)

        for item in post_dir.iterdir():
            if item.is_file() and item.name != "index.md":
                shutil.copy2(item, target_post_dir)

        with open(source_file, 'r', encoding='utf-8') as f:
            content = f.read()

        parts = content.split('+++', 2)
        if len(parts) < 3:
            print(f"  ! Warning: Could not parse front matter for {source_file}. Skipping.")
            continue
        
        front_matter_str, markdown_content = parts[1], parts[2]
        front_matter = toml.loads(front_matter_str)
        source_lang = front_matter.get("original_language", "en")
        print(f"  > Original language: {source_lang}")

        # --- Batch process front matter fields ---
        text_to_process = f"TITLE: {front_matter.get('title', '')}\n---\nDESCRIPTION: {front_matter.get('description', '')}"
        processed_text = get_ai_response(text_to_process, source_lang, TARGET_LANG)
        
        if processed_text:
            title_match = re.search(r"TITLE:\s*(.*)", processed_text)
            desc_match = re.search(r"DESCRIPTION:\s*(.*)", processed_text, re.DOTALL)
            if title_match:
                front_matter["title"] = title_match.group(1).strip()
            if desc_match:
                front_matter["description"] = desc_match.group(1).strip()
        
        # --- Batch process markdown content ---
        content_parts = []
        text_batch = []
        
        # Split content by code blocks to isolate translatable text
        # The regex splits the text, keeping the code blocks as delimiters
        split_content = re.split('(```[\s\S]*?```)', markdown_content)

        for i, part in enumerate(split_content):
            if i % 2 == 1: # This is a code block (the delimiter)
                content_parts.append(part)
            else: # This is regular markdown text
                if part.strip():
                    content_parts.append(part) # Add the text part

        # Now, let's create a single large string to send to the AI
        full_text_to_process = ""
        placeholder_map = {}
        placeholder_index = 0

        for part in content_parts:
            if part.startswith("```"):
                full_text_to_process += f"\nCODE_BLOCK_{placeholder_index}\n"
                placeholder_map[f"CODE_BLOCK_{placeholder_index}"] = part
                placeholder_index += 1
            else:
                full_text_to_process += part
        
        processed_content_full = get_ai_response(full_text_to_process, source_lang, TARGET_LANG)
        
        # Reconstruct the content from the AI response
        if processed_content_full:
            final_markdown = processed_content_full
            for placeholder, code_block in placeholder_map.items():
                final_markdown = final_markdown.replace(placeholder, code_block, 1)
        else:
            final_markdown = markdown_content # Fallback to original content on error
        
        # Rebuild and Save
        new_front_matter_str = toml.dumps(front_matter)
        new_content = f"+++\n{new_front_matter_str}+++\n{final_markdown}"
        
        target_file_path = target_post_dir / "index.md"
        with open(target_file_path, 'w', encoding='utf-8') as f:
            f.write(new_content)
        
        print(f"  ✓ Successfully created: {target_file_path}")


if __name__ == "__main__":
    process_posts()

Of course. Protecting mathematical formulas is a critical requirement for any technical blog. The best way to handle this is to treat LaTeX blocks just like you treat code blocks: as “protected” content that should never be sent to the AI.

We can achieve this by modifying the script to use a placeholder system. Before sending the text to Ollama, the script will find all code blocks and all LaTeX formulas, replace them with unique placeholders (e.g., __PROTECTED_BLOCK_0__), and store the original content. After the AI translates the remaining text, the script will substitute the placeholders back with the original, untouched code and formulas.

This approach is robust and guarantees that your formulas and code remain perfect.

Key Changes in the Script

  1. Combined Regex: A powerful regular expression will find all content to be protected: code blocks and all common LaTeX delimiters ($...$, $$...$$, \(...\), \[...\]).
  2. Placeholder System: Instead of complex splitting, we’ll replace protected content with placeholders before the API call and restore it after.
  3. Updated AI Prompt: The instructions for the AI will be updated to explicitly tell it to ignore the placeholders.

The Final, Updated Script for Ollama

import shutil
import requests
import toml
import re
from pathlib import Path

# --- CONFIGURATION for OLLAMA ---
API_URL = "http://localhost:11434/v1/chat/completions" # Ollama's local endpoint
AI_MODEL = "llama3" # Your local model

SOURCE_DIR = Path("content/original/posts")
TARGET_LANG = "en"
TARGET_ROOT_DIR = Path(f"content/{TARGET_LANG}/posts")

# --- HELPER FUNCTION: THE AI CALL (Modified for Placeholders) ---
def get_ai_response(text, source_lang, target_lang):
    """
    Sends a chunk of text to the local Ollama model for translation or enhancement.
    """
    if not text.strip():
        return ""

    if source_lang == target_lang:
        task_instruction = "You are an AI assistant for a technical blog. Enhance the following text for clarity and style. Fix any grammatical errors."
    else:
        task_instruction = f"You are a professional translator for a technical blog. Translate the following text from '{source_lang}' to '{target_lang}'."

    prompt = f"""

    IMPORTANT RULES:
    1.  Preserve all Markdown formatting (headings, lists, etc.).
    2.  DO NOT translate or change URLs or image paths.
    3.  DO NOT translate text inside single backticks (`like_this`).
    4.  The text may contain placeholders like `__PROTECTED_BLOCK_0__`. DO NOT translate, change, or remove these placeholders. Keep them exactly as they appear.
    5.  Your output MUST be ONLY the processed text, with no extra explanations or preambles.

    Here is the text to process:
    ---
    {text}
    """

        ai_content = result['choices'][0]['message']['content'].strip()
        print("  > Received processed text from Ollama.")
        return ai_content
    except requests.exceptions.RequestException as e:
        print(f"  ! Error calling Ollama API: {e}")
        print("  ! Is the Ollama application running? Check with 'ollama list' in your terminal.")

# --- CORE SCRIPT LOGIC (With Placeholder System) ---
def process_posts():
    if not SOURCE_DIR.exists():
        print(f"Source directory not found: {SOURCE_DIR}")
        return

    # This regex finds code blocks AND all common LaTeX delimiters.
    # It correctly handles inline ($...$) and display ($$..$$, \[...\]) math.
    PROTECTED_BLOCK_REGEX = re.compile(
        r"(```[\s\S]*?```|"  # Code blocks
        r"\$\$[\s\S]*?\$\$|"  # Display math $$...$$
        r"\\\[[\s\S]*?\\\]|"  # Display math \[...\]
        r"\$[^$]*?\$|"      # Inline math $...$
        r"\\\(.*?\\\))",    # Inline math \(...\)
        re.MULTILINE
    )

    print(f"Scanning for posts in {SOURCE_DIR}...")

    for post_dir in [d for d in SOURCE_DIR.iterdir() if d.is_dir()]:
        source_file = post_dir / "index.md"
        if not source_file.exists(): continue

        print(f"\n--- Processing: {post_dir.name} ---")

        # 1. Setup destination and copy assets
        target_post_dir = TARGET_ROOT_DIR / post_dir.relative_to(SOURCE_DIR)
        target_post_dir.mkdir(parents=True, exist_ok=True)
        for item in post_dir.iterdir():
            if item.is_file() and item.name != "index.md":
                shutil.copy2(item, target_post_dir)

        # 2. Read and parse front matter
        with open(source_file, 'r', encoding='utf-8') as f:
            content = f.read()
        parts = content.split('+++', 2)
        if len(parts) < 3: continue
        
        front_matter_str, markdown_content = parts[1], parts[2]
        front_matter = toml.loads(front_matter_str)
        source_lang = front_matter.get("original_language", "en")
        print(f"  > Original language: {source_lang}")

        # 3. Process front matter (title and description)
        text_to_process = f"TITLE: {front_matter.get('title', '')}\n---\nDESCRIPTION: {front_matter.get('description', '')}"
        processed_text = get_ai_response(text_to_process, source_lang, TARGET_LANG)
        if processed_text:
            title_match = re.search(r"TITLE:\s*(.*)", processed_text)
            desc_match = re.search(r"DESCRIPTION:\s*(.*)", processed_text, re.DOTALL)
            if title_match: front_matter["title"] = title_match.group(1).strip()
            if desc_match: front_matter["description"] = desc_match.group(1).strip()
        
        # 4. Protect and process the main markdown content
        placeholder_map = {}
        placeholder_index = 0

        def protect_block(match):
            nonlocal placeholder_index
            placeholder = f"__PROTECTED_BLOCK_{placeholder_index}__"
            placeholder_map[placeholder] = match.group(0)
            placeholder_index += 1
            return placeholder

        # Replace all protected blocks (code AND math) with placeholders
        text_with_placeholders = PROTECTED_BLOCK_REGEX.sub(protect_block, markdown_content)

        # Send only the "clean" text to the AI
        processed_content = get_ai_response(text_with_placeholders, source_lang, TARGET_LANG)
        
        # Restore the protected blocks
        if processed_content:
            final_markdown = processed_content
            for placeholder, original_block in placeholder_map.items():
                final_markdown = final_markdown.replace(placeholder, original_block, 1)
        else:
            final_markdown = markdown_content # Fallback on error

        # 5. Rebuild and save the new file
        new_front_matter_str = toml.dumps(front_matter)
        new_content = f"+++\n{new_front_matter_str}+++\n{final_markdown}"
        
        target_file_path = target_post_dir / "index.md"
        with open(target_file_path, 'w', encoding='utf-8') as f:
            f.write(new_content)
        
        print(f"  ✓ Successfully created: {target_file_path}")

if __name__ == "__main__":
    process_posts()

The process is exactly the same as before:
