Git Basics

Find Version

(Often used to test if git is installed)

git --version

Set username and email address

git config --global user.name "jokobus"
git config --global user.email "jokobus@example.com"

If you want to use the username only for the git repository of the current directory, remove the global option.

If you want to see which username and email is currently set, use

git config --global user.name
git config --global user.email

If you are using a credential manager, you might need to clear the stored credentials

git credential-cache exit

Init

Initiate empty git repository (in your directory). Afterwards one can add/remove/edit files/folders.

git init

Status

Status Gives an overview over tracked and untracked entities.

git status

Stage

Add (=stage) untracked files to git

git add <filename>
git add --all   # add all (incl. deleting files)
git add -A      # shorthand for --all
git add .       # Will add all modifications and new files/folders, but not delete files/folders

Commit

git commit -m "Some message"
git commit -a -m "Some message" # -a commits all changes directly, skipping stage (for small changes, not recommended), or use -am

Log & Help

Git log

git log

Git help

git command -help   # help for a command
git --help          # general git help
git help --all      # very, very long list

Branches

Create a branch (~copy) of the main branch, edit your branch and merge it with the main branch.

git branch mybranch      # create new branch
git branch                      # verify in which branch you are (* main)
git checkout mybranch    # therefore switch to new branch

# alternative: create and move to new branch in one line
git checkout -b mybranch

After making changes, add, commit and merge.

git add -A
git commit -m "Some message"    # Commit to mybranch
git checkout master             # Switch back to master
git merge mybranch
git branch -d mybranch   # Delete temporary branch after successful merge

If the merge fails due to a conflict, edit the file and commit again.

Use Git with Github

Important: Stay up to date!! Otherwise you and your teammembers might work on different versions and merging your branches can become a nightmare.

Make a repository available locally

  • Remote add: Upload content of an existing git repository to a new github repository.

    1. Log in to github and create a new (empty) repository
    2. cd into your local directory
    3. Add origin and push your local files to github
    git remote add origin https://github.com/jokobus/hello-world.git
git push --set-upstream origin master   # push current main branch to origin url and set it as default for the remote branch

  • Or Clone an existing repository

    git clone https://github.com/jokobus.hello-world.git

  • Difference between git remote add origin and git clone

    • remote add origin creates a remote connection to a repository
    • clone copies a github repository, you cannot push directly

Useful commands

git fetch origin        # download latest version from github
git pull origin         # pull = fetch + merge
git status              # show status of current repositiory
git branch -a           # -a show local and remote repositories, -r for remote only
git log origin/main     # show the log
git diff origin/main    # show differences between current branch and main

You can also create a new branch directly on https://github.com/yourrepository and then pull again, change to that branch and edit locally.

Workflow

Your own repository

  • Create a new Branch
  • Make changes and add Commits
  • Open a Pull Request
  • Review
  • Deploy
  • Merge

Someone else’s repository

  • Fork a repo on Github
  • Clone your fork locally (will be called origin)
    mkdir C:/Users/me/wherever/myproject # create a new directory
cd C:/Users/me/wherever/myproject    # enter that repository
git init # initialize git in 
git clone https://github.com/yourusername/my-forked-repo.git # clone the repository
git pull origin main # pull the content
  • Add the original repo as upstream (into the same directory)
    git remote add upstream https://github.com/someoneelsesusername/original-repository.git
git pull upstream main
  • Work on your fork and commit all your changes there
  • Pull updates from the original repository
  • When done, create a pull request to the original repository

Further useful commands

git remote -v # See version
git remote rename origin upstream # rename from origin to upstream
git

Pull Request

Create pull Requests

A pull request is a request to merge your fork with the original. Somebody (else or you yourself) has to review and verify the request.

  • Go to your fork on Github
  • Click on pull request (above the code)
  • Add a message and send

Review pull Requests

(If you have the necessary rights)

  • On the Github webpage go to the tab Pull requests
  • Select the pull request
  • Add a comment and merge

.gitignore

Create a .gitignore file in your repository. In the .gitignore file you can specify what parts of a project won’t be shared with others, i.a.

  • log files
  • temporary files
  • hidden files
  • personal files

Note:

  • You can create a .gitignore file in a subdirectory; .gitignore files only apply down the directory tree.
  • The .gitignore though will always be shared.
  • You specify ignores in the .git/info/exclude file. Works as .gitignore, but won’t be shown anyone else

Example: In the .gitignore file add:

# ignore ALL .log files except main.log
*.log
!main.log

# ignore ALL files in ANY directory named temps
temp/

Useful options to put into the .gitignore: https://www.w3schools.com/git/git_ignore.asp?remote=github

SSH

If you want to use SSH instead of HTTPS.

  1. Create a keypair
    1. Create a keypair with your email address
      ssh-keygen -t rsa -b 4096 -C "jokobus@example.com"
    2. Enter directory where to save the key (e.g. C:/Users/user/.ssh/id_rsa)
    3. Enter a passphrase (twice), secure this passphrase in your password manager
    4. Add the SSH key pair to the SSH-Agent
      ssh-add C:/Users/user/.ssh/id_rsa
  2. Add public key to Github
    1. Go to the Github webpage
    2. In your profile settings select SSH and GPG keys
    3. Click the New SSH key button, select a title and paste the key into the key field
    4. Test connection in shell
      ssh -T git@github.com
      Now the console prints some text. If the last line contains your github username, you’re authenticated.
  3. Add new Github SSH remote
    1. On the Github site in your repository click the green Code button.
    2. Select SSH and copy the link
    3. Add the repository (replace git@github… with the link)
    git remote add ssh-origin git@github.com:myusername/my-repository.git
    1. Now you can edit your repository via SSH!

Note: You can change a remote origin from HTTPS to SSH with:

git remote set-url origin git@github:myusername/my-repository.git

Git Revert

We use the revert command to undo a previous commit, while keeping the log intact.

  1. Find the commit you want to undo
    git log --oneline
  2. Revert the change
    • If you only want to revert the last commit
      git revert HEAD --no-edit   # --no-edit option automatically commits
    • If you want to go back more commits
      git revert HEAD~x   # x is number of commits to go back (count in log)

Git Reset

We use reset to move the repository back to a previous commit, discarding any changes made after that commit. The log will be interrupted.

Caution: Use the reset command with caution, especially if you work together with others on the same repository.

  1. Find the commit to whose version (the version before you messed up) you want to return to in the log
  2. Copy the first seven characers (the commit hash, eg. 12a34bc)
  3. Reset
    git reset 12a34bc
  4. Check the log again

Git Amend

commit –amend is used to modify the most recent commit. The amended commit replaces the last commit entirely.

Modify the commit message First, commit some changes with a message. Afterwards you can change the message by:

git commit --amend -m "New message for last commit. "

Disconnect from a Github repository

In case you have cloned a Github repository to a directory, but want to remove it again, follow these steps.

git remote remove origin    # or upstream etc.
git remote -v               # if nothing is printed, your disconnected

Then delete the directory.
