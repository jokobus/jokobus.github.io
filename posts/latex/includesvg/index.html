How to include Scalable Vector Graphics (svg) in $\LaTeX$

Guide how to use \includesvg[ ] (image.svg) in $\LaTeX$

  • Howto
  • Howto
  • LaTeX
2024-07-17
2 minutes to read

Disclaimer: This post was created using AI.

You may know the popular LaTeX package graphicx which you can use as:

\includegraphics[options]{filename}

graphicx does not support svg, but there is the package called svg that does the job.

How to use the svg package:

  1. Install Inkscape

    On Windows you can use winget.

    winget install -e --id Inkscape.Inkscape

  2. Copy your image.svg file into your document root folder (= right next to your .tex file)

  3. This is a MWE for your .tex file:

    \documentclass{article}
\includepackage{svg}
\begin{document}
    \begin{figure}
        \includesvg[]{image}
    \end{figure}
\end{document}

Notes:

  • File ending is optional (image or image.svg both work)
  • There are many parameters one can set in the square brackets (e.g. width=1cm, height=1cm, scale=0.5, and many more)
  • Download and read the svg package documentation on CTAN for more information

Oh no!

It would have been too easy. If you click on Build, you will run into an error:

example.tex: error: 120: Package svg Error: File `image_svg-tex.pdf' is missing. \includesvg{image}

You can solve this error by opening (in TeXstudio) Settings>Commands and adding the flag

--shell-escape

to your PdfLaTeX run command. The command now should look like:

pdflatex.exe -synctex=1 -interaction=nonstopmode --shell-escape %.tex

  • If you are using XeLaTeX, LulaTeX, etc. you should add the flag there (probably, I didn’t test it).

  • If you are using Texmaker, the whole idea is the same, but the exact settings may be somewhere else.

Save the configuration and run again.

More oh no!

We are not there yet. We need to solve the error message

example.tex: error: 119: Package svg Error: Inkscape version not detected. ...ncludesvg{image}

This error means that LaTeX doesn’t know where to find Inkscape. Therefore we add the path whereever inkscape was installed as a custom command. To do so, first locate the right folder (on Windows the default is: C:\Program Files\Inkscape\bin) then open Settings>Build and add the path to Commands.

That’s it, now it should run.
