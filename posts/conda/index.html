Why Conda?

Conda is an environment and package management system, widely used in scientific computing and data science. Very handy if you program in Python. Conda features:

  • Cross-Platform Compatibility
  • Package Management
  • Environment Management (you can switch easily)
  • Version control
  • Dependency Management

The handy thing about conda is that you can create a new environment for each project that you have. Then you install all the packages for each project in it’s own dedicated environment. Then you can swap back and forth to another project that might need totally different packages and versions. The different packages and versions are kind of ‘containerized’ and don’t interfer with each other.

Installation

You can install Anaconda (GUI) or miniconda (minimalistic). However, I recommend the community-driven miniforge ‘version’.

Download the Miniforge Installer from the conda-forge website or use

wget https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh

Then run the script

bash Miniforge3-Linux-x86_64.sh -p /path/where/to/install/miniforge

This will add the path to PATH. If interested, take a look at the ~/.bashrc file.

Test Conda is installed

conda info
conda --version
conda --help        # To get an overview

Customization

You can skip this chapter, if you don’t want to customize the installation. In most cases the default is all you need.

If you want to manually specify where conda should create new environments and store packages, write it into a .condarc file.

Depending on the scope you want this file to have you can either put it into a project folder or place it under root. For global:

nano ~/.condarc

Paste

envs_dirs: 
  - /path/to/your/custom/environments
pkgs_dirs: 
  - /path/to/your/custo/packages

You can also use specify channels there, just append the following:

channels: 
  - conda-forge

conda-forge is the (only) default channel.

To verify your configuration:

conda config --show

Environments

List Environments

To see which environments are installed let conda list them

conda env list

Create a new environment

Environments are handy! You can create a new environment for each project you have and each environment will be independent from the other environments. E.g. you can download for a specific project python 3.9 and matplotlib and for another python 13.3 and tensorflow. Just create two different environments and install all the packages. See the introducion why Conda is useful.

You can create a new environment (here called my_environment) with one of the following commands:

conda create -n my_environment               # Specify -n or --name
conda create -n my_environment python=3.9    # Specify python version

There are a bunch of possibilites how to specify the version, for both the python version and the package version.

You can also clone an already existing environment.

conda create --clone my_environment --name my_environment2

It is also possible to specify where the environment should be created (environment will be called according to the deepest subfolder, in this case my_environment).

conda create -p /path/to/where/to/install/my_environment

Furthermore, you can save an existing environment to a .txt file

conda list --explicit > saved_environment.txt

or create an environment from such a .txt file.

conda create --name my_environment saved_environment.txt

You can delete an environment and all it’s installed packages

conda env remove --name my_environment

Enter/Leave an environment

In your console you see on the very left of each input line in which environment you currently are. E.g. on Linux

(base) jokobus@MyComputer:-$
(my_environment) jokobus@MyComputer:-$

and Windows

(base) PS C:\Users\jokobus>
(my_environment) PS C:\Users\jokobus>

You can activate an environment

conda activate my_environment

and switch back to the base environment.

conda deactivate

Disable Base Environment

You can permanentely disable the base environment with:

conda config --set auto_activate_base false

Rename/Delete an environment

You can rename a conda environment with

conda rename --name old_name new_name

An alternative (the old) approach is to clone the repository and remove the original. Create the clone

conda cerate --name new_name --clone old_name

Remove the environment

conda remove --name old_name --all

Update/Install specific python version

If you have forgotten to install the right python version at the creation of the environment or want to update, then you can install a different python version in your existing conda environment. Just keep in mind that the packages need to be compatible with the new python version. First, let’s see which python version you are currently using.

python --version

You can update python to the latest version

conda install python

Alternatively you can install a specific python version (e.g. 3.12) with

conda install python=3.12

After installing a new python version I recommend to conda list all packages and see that all packages you need are still there. Also it might be a good idea to update the packages or reinstall them if they were removed. Continue reading for more info how to do it.

Packages

Install/Uninstall packages

Now comes the interesting part. The Installation of packages!

It is as simple as

conda install mypackagename

You can even stack the packages.

conda install mypackagename1 mypackagename2 mypackagename3

You can also specify the version you want to use.

conda install mypackage=3.12

If you want to uninstall a package

conda remove mypackage

Update installed packages

You can update a package

conda update mypackage

or install (update/downgrade) a specific version.

conda install mypackage=1.1.1

or update all packages

conda update --all

List installed packages

You can list the packages that are installed in your current environment

conda list

That’s basically all you need to know for using conda. Have fun coding - conda will help you with the necessary package management.
