tl;dr SQL let’s you access and manipulate databases.

Why SQL

SQL can execute queries, retrieve data, insert records, update records, delete record to/from a database as well as create/delete a database, create/delete tables in a database, create views and stored procedures in a database and finally set permissions on tables, procedures and views.

Many database programs build on SQL, but add some (proprietary) functionalities.

If you want to embedd database information via SQL on a website you use a database program such as MySQL. Besides you will need a server-side scripting language (PHP) and HTML and CSS for the frontend.

Basics

  • Column = A Feature or observation criteria
  • Row = A Record or observation

Tipp: SQL keywords are not case sensitive

SELECT column1, column2 FROM mytable WHERE condition; 
SELECT * FROM mytable;
select column1, column2 from mytable;

SELECT DISCTINCT

SELECT DISTINCT returns only distinct (non-duplicate) values

SELECT DISTINCT column1 FROM mytable;

You can combine it with the COUNT argument, to count how many distinct entries are there, e.g. in the column Country.

SELECT COUNT (DISTINCT Country) FROM mytable;

ORDER BY

Use the ORDER BY statement to order the order of the rows, e.g. py ascending price.

SELECT column1, column2 FROM mytable ORDER BY column1, column2 ASC|DESC;
SELECT ProductName, Price FROM mytable WHERE Price>=50 ORDER BY Price DESC, ProductName ASC;

WHERE

Appending WHERE you can filter the returned entities. E.g. the following example only returns records where the Country feature is Germany. (Of course the table has to have columns called ‘Country’ and ‘Revenue’. ) Use single quotes for Text / “Strings”.

SELECT * FROM mytable WHERE Country='Germany';
SELECT * FROM mytable WHERE Revenue > 100000;

AND, OR, NOT operator

The WHERE clause can contain one or multiple AND, OR and NOT operators. AND and OR are rather self explaining. NOT is used with other operators to invert their result. You can combine multiple operators and use brackets for semantics.

SELECT column1, column2, FROM mytable WHERE condition1 AND (condition2 OR condition3) AND NOT condition4;
SELECT * FROM mytable WHERE Country='Germany' AND Revenue > 1000000;

Some Examples: First, select all entries from countries that start with G (Gambia, Germany, Greece, etc.) or F (Finnland, France, Fidji, etc.) and where the revenue is greater than 100.

SELECT * FROM mytable WHERE Revenue > 100 AND Country LIKE 'G%' OR Country LIKE 'F%'; 
SELECT * FROM mytable WHERE Country IN ('Germany', 'France');

Example: All entrise with a revenue that is not between specific boundaries shall be returned.

SELECT * FROM mytable WHERE Revenue NOT BETWEEN 100 AND 1000;

INSERT INTO

INSERT INTO is used to insert new records in a table. There are two different ways to write it:

  • By using the first command you can specify only specific columns, the rest is left blank (null value).
    INSERT INTO mytable (column1, column2, column3) VALUES (value1, value2, value3);
  • And with this command you can enter multiple lines (here it’s three) with one call.
    INSERT INTO mytable VALUES (value11, value12, value13), (vaule21, value22, value23), (value31, value32, value33);

NULL Value

A field with a NULL is a field without a value. You can use the IS NULL and IS NOT NULL operators to check for nulls. Something like column1 == NULL does not work.

SELECT * FROM mytable WHERE column1 IS NULL;
SELECT * FROM mytable WHERE column1 IS NOT NULL;

UPDATE SET

Use the UPDATE to modify an existing record. Be careful that the WHERE condition is exact (e.g. use some distinct ID)!

UPDATE mytable SET column1 = value1, column2 = value2 WHERE condition;

DELETE

Of course it is also possible to delete records again. To clear a table, just ommit the WHERE condition.

DELETE FROM mytable WHERE condition;

DROP TABLE (Delete Table)

To delete a table again, use

DROP TABLE mytable;

SELECT TOP

The SELECT TOP clause is used to specify the number of records to return. (E.g. the table has 100 records but you want just the first three returned. )

Note: This command may differ for different database systems.

SELECT TOP 3 * FROM mytable;
SELECT TOP 50 PERCENT * FROM mytables;

In MySQL for example the syntax is

SELECT * FROM mytable LIMIT 3;

Aggregate Functions

An aggregate function performs a calculation on a set of values and returns a single value. They are often used together with GROUP BY which splits the return into groups first. Aggregate functions ignore null values.

Aggregate FunctionDescription
MIN()returns the smallest value within the selected column
MAX()returns the largest value within the selected column
COUNT()returns the number of rows in a set
SUM()retruns the total sum of a numerical column
AVG()returns the average value of a numerical column

MIN() and MAX()

SELECT MIN(columnname) FROM mytable WHERE condition;
SELECT MIN(Price) FROM mytable;
SELECT MAX(Price) FROM mytable;

AS

When using MIN() or MAX() the returned column does not have a descriptive name, but with the AS keyword one can set a name.

SELECT MIN(Price) AS SmallestPrice FROM Products;

COUNT()

Use COUNT to count the number of rows that match a specific condition. If you ommit WHERE and use a column without NULLs (e.g. an ID column) this will effectively return the length of the table. Add DISTINCT to don’t count duplicates in.

SELECT COUNT(columnname) FROM mytable WHERE condition; 
SELECT COUNT(DISTINCT columnname) FROM mytable WHERE condition;

You can even use an ALIAS to name the returned column “Number of rows”

SELECT COUNT(*) AS [Number of rows] FROM mytable;

Often it is useful to combine COUNT and GROUP BY. This example will return how many entries are there for each country.

SELECT COUNT(*) FROM mytable GROUP BY Country;

SUM()

The SUM clause is used to sum all entries in a numeric column.

SELECT SUM(Price) AS [Total Cost] FROM mytable;
SELECT OrderID, SUM(Price) AS [Cost per Order] FROM mytable GROUP BY OrderID;

AVG()

The AVG clause takes the average of a numeric column. You can use it e.g. to find all rows where with a value greater than the average.

SELECT * FROM mytable WHERE Price > (SELECT AVG(Price) FROM mytable);

LIKE

The LIKE operator is used in a WHERE clause to search for a specified pattern in a column, often in combination with a Wildcard.

  • The percent sign % represents zero, one or multiple characters
  • The underscore sign _ represents one and only one single character

E.g. select all rows where the country’s name starts with G. % is a wildcard for th rest of the name (otherwise SQL would only return countries whose whole name is ‘G’).

SELECT column1, column2 FROM mytable WHERE column1 LIKE pattern;
SELECT * FROM mytable WHERE Country LIKE 'G%';
