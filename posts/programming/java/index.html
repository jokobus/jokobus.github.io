3. Methods

Basic example

Methods must always be declared inside a class.

public class Main {
  static void myMethod(String fname, String lname, int age) {
    System.out.println(fname + " " + lname + " is " + age);
  }
  static int myAddition(int a, int b) {
	return a + b;
  }
	
  public static void main(String[] args) {
    myMethod("Liam", "Miller", 5);
    myMethod("Jenny","Barker", 8);
    myMethod("Anja", "Meier", 31);
	myMethod("Peter", "the Pointer", 0);
	
	int sum = myAddition(5,3); //5+3=8
  }
}

Method overloading

You can declare multiple methods with the same name, but different parameters. Depending on the provided parameters the consequent method will be called.

int myMethod(int x)
float myMethod(float x)
double myMethod(double x, double y)

Hint: Widening casting will be done automatically, but narrowing casting not.

4. Classes

A class is a template for objects, and an object is an instance of a class. When the individual objects are created, they inherit all the variables and methods from the class.

public class Main {
	int x = 0;

	public static void main(String[] args) {
		Main myObj1 = new Main();
		Main myObj2 = new Main();
		myObj1.x = 5;
		myObj2.x = 3;
		System.out.println(myObj1.x); // 5
		System.out.println(myObj2.x); // 3
	}
}

Multiple Classes

Each java class has to be stored in a file with the name of that class. So the Main class is stored in a file called Main.java and the class Second has to be stored in a different file called Second.java. One needs to compile both.

static vs public functions

  • static functions can be called without creating an object (but creating an object is also okay)
  • public functions must be called by creating objects
public class Main() {
    static void myStaticFunction(){
        // do …
    }
    public void myPublicFunction(){
        // do …
    }
    public static void main(String[] args) {
        myStaticFunction();
        // myPublicFunction(); // would throw a compile error
        Main myObj = new Main();
        myObj.myPublicFunction();
    }
}

Constructor

  • The constructor is used to generate a new object.
  • The constructor is autogenerated by java.
  • If you explicitely write down the constructor, you can specify parameters (inputs at constrution and default values)
public class Main(){
    // This is the constructor
    public Main(int y){
        int x = y;
    }
    
    public static void main(String[] args){
        Main myObj = new Main(5);
        System.out.println(myObj.x); //5
    }
}

Modifiers

Access-Modifiers

Access-Modifiers modifies access level of a class.

For classes

  • public: The class is accessible by another class.
  • default: The class is only accessible by other classes in the same package.

For attributes, functions & constructors

  • public: The code is accessible for all classes.
  • private: The code is only accessible within the declared class.
  • default: The code is only accessible in the same package.
  • protected: The code is accessible in the same package and subclass.

Non-access Modifiers

Non-Access Modifiers do not control access level, but provide other functionality.

For classes

  • final: The class cannot be inherited by other classes.
  • abstract: The class cannot be used to create objects. (To access an abstract class, it must be inherited from another class.)

For attributes and methods

  • final: Attributes and methods cannot be overriden/modified. (final int x = 5; x = 10; // Compile error)
  • static: Attributes and methods belongs to the class, rather than an object.
  • abstract: Can only be used in an abstract class, and can only be used on methods. The method does not have a body, for example abstract void run();. The body is provided by subclass (inherited from).
  • transient: Attributes and methods are skipped when serializing the object containing them.
  • synchronized: Methods can only be accessed by one thread at a time
  • volatile: The value of an attribute is not cached thread-locally, and is always read from the “main memory”.

Encapsulation

  • Method to ensure that sensitive data cannot be read by any class.
  • Declare variables as private
  • Write a public get-method which returns the variable value, and a public set-method which sets the value.

Inheritance (Subclass and Superclass)

handy for code reusability: reuse attributes and methods of an existing class when creating a new class.

// Superclass
class Vehicle {
  protected String brand = "BMW"; 
  // …
}
// Subclass inherits Attributes/Methods from Superclass
class Car extends Vehicle {
  private String modelName = "X6";
  public static void main(String[] args) {
    Car myCar = new Car();
    System.out.println("I have a "+myCar.brand+" "+myCar.modelName);
  }
}

If you don’t want a class to be inheritable, use the final keyword, e.g. final class Vehicle.

Polymorphism

The following example is very interesting. It uses inheritance and method overloading.

class Animal {
  public void animalSound() {
    System.out.println("The animal makes a sound");
  }
}

class Cat extends Animal {
  public void animalSound() {
    System.out.println("The cat says: miau");
  }
}

class Dog extends Animal {
  public void animalSound() {
    System.out.println("The dog says: wau");
  }
}

class Main {
  public static void main(String[] args) {
    Animal myAnimal = new Animal();  // Create a Animal object
    Animal myCat = new Cat();  // Create a Pig object
    Animal myDog = new Dog();  // Create a Dog object
    myAnimal.animalSound(); // The animal makes a sound
    myCat.animalSound(); // The cat says: miau
    myDog.animalSound(); // The dog says: wau
  }
}

Nested classes

class OuterClass {
  int x = 1;
  class InnerClass{
    int y = 2;
  }
}
public class Main {
  public static void main(String[] args) {
    OuterClass myOuterObject = new OuterClass(); // New outer object
    OuterClass.InnerClass myInnerObject = new OuterClass.InnerClass(); // New inner object
    System.out.println(myOuterObject.x + myInnerObject.y)
  }
}
  • If you declare the InnerClass as private, an outside class can’t access it.
  • When you use static, you don’t need to create an inner object.

Abstract class

Abstraction is the process of hiding certain details and showing only essential information to the user.

  • Abstract class: cannot be used to create objects, must be inherited
  • Abstract method: can only be used in an abstract class, doesn’t have a body. The body is provided by the subclass.
abstract class Animal {
  public abstract void animalSound(); // does not have a body
}

class Cat extends Animal {
  public void animalSound() {
    System.out.println("The cat says: miau");
  }
}

class Main {
  public static void main(String[] args) {
    // Animal myAnimal = new Animal();  // Create a Animal object not possible
    Animal myCat = new Cat();  // Create a Pig object
    // myAnimal.animalSound(); // Execute the abstract function is not possible
    myCat.animalSound(); // The cat says: miau
  }
}

Interfaces

Notes on Interfaces from w3schools .

  • Like abstract classes, interfaces cannot be used to create objects (in the example above, it is not possible to create an “Animal” object in the MyMainClass)
  • Interface methods do not have a body - the body is provided by the “implement” class
  • On implementation of an interface, you must override all of its methods
  • Interface methods are by default abstract and public
  • Interface attributes are by default public, static and final
  • An interface cannot contain a constructor (as it cannot be used to create objects)

Why And When To Use Interfaces?

  1. To achieve security - hide certain details and only show the important details of an object (interface).

  2. Java does not support “multiple inheritance” (a class can only inherit from one superclass). However, it can be achieved with interfaces, because the class can implement multiple interfaces. Note: To implement multiple interfaces, separate them with a comma (see example below).

interface FirstInterface {
  public void myMethod(); // interface method, no body
}

interface SecondInterface {
  public void myOtherMethod(); // interface method, no body
}

class DemoClass implements FirstInterface, SecondInterface {
  public void myMethod() {
    System.out.println("Some text…");
  }
  public void myOtherMethod() {
    System.out.println("Some other text…");
  }
}

class Main {
  public static void main(String[] args) {
    DemoClass myObj = new DemoClass();
    myObj.myMethod();
    myObj.myOtherMethod();
  }
}

Enums

An enum represents a group of constants (public, final, static variables). Used for known, fix values, such as month names, weekdays, colors, etc. Handy if combined with the switch statement.

public class Main {
  enum Level {
    LOW, 
    MEDIUM,
    HIGH
  }
  public static void main(String[] args){
    Level myVar = Level.MEDIUM;
    System.out.println(myVar); //MEDIUM

    for (Level myVar : Level.values()){
      System.out.print(myVar+", "); //LOW, MEDIUM, HIGH, 
    }
  }
}

5. Java Data Structures

ArrayList

ArrayList class is a resizable arry in the java.util package (normal arrays are not resizable), where you can add/remove elements on the go.

Use Collections in the java.util package to sort lists. Will sort by default in ascending order. Use option Collections.reverseOrder() for descending order.

import java.util.ArrayList;
import java.util.Collections; // Used to sort the ArrayList (alphabetically, ascending, etc. )

public class Main {
  public static void main(String[] args){
    ArrayList<String> animals = new ArrayList<String>(); // Create new ArrayList object storing Strings
    // Instead of <String> one can use <Boolean>, <Character>, <Double>, <Integer>, etc. 
    animals.add("Zebra");
    animals.add("Mephitidae");
    animals.add("Cat")
    System.out.println(animals); // [Zebra, Mephitidae, Cat]
    animals.add(2,"Elephant"); // Specify position, start with 0
    System.out.println(animals); // [Zebra, Mephitidae, Elephant, Cat]
    System.out.println(animals.get(2)); // Elephant
    animals.set(2,"Dog"); // Overwrite Elephant
    animals.size(); 
    for (int i = 0; i < animals.size(); i++){
      // do …
    }
    Collections.sort(animals);
    System.out.println(animals); // [Cat, Dog, Mephitidae, Zebra]
    Collections.sort(animals, Collections.reverseOrder()); // Reverse the order
    System.out.println(animals); // [Zebra, Mephitidae, Dog, Cat]
    animals.remove(1); // Delete Mephitidae
    animals.clear(); // Empty ArrayList
  }
}

HashMap

Hash maps are key/value pairs of same or different type, e.g. HashMap<String, String>, HashMap<String, boolean>, Hashmap<int, double> etc.

import java.util.HashMap();
public class Main{
  public static void main(String[] args){
    HashMap<String, String> Capitals = new HashMap<String, String>();
    Capitals.put("Germany", "Berlin");
    Capitals.put("England", "London");
    Capitals.put("France", "Paris");
    Capitals.put("Austria", "Vienna");
    System.out.println(Capitals); // {Germany=Berlin, England=London, France=Paris}
    System.out.println(Capitals.get("England")); // London
    System.out.println(Capitals.remove("Austria")); 
    System.out.println(Capitals.size()); // 3
    for (String myCountry : Capitals.keySet()){
      System.out.println(myCountry); // Germany, England, France
      System.out.println(myCountry+" is in "+myCountry.get(myCountry));
    }
    for (String myCapital : Capitals.value()){
      System.out.println(myCapital); // Berlin, London, Paris
    }
  }
}

Wrapper Classes

Wrapper classes provide a way to use primitive variables as objects. The first character of the type classifier is written Uppercase, e.g. Char, Int, Double, etc. Useful if you work with arrays, arraylists where you can only store objects.

// ArrayList<int> mylist = new Arraylist<int>(); // Error
ArrayList<Integer> mylist = new ArrayList<Integer>(); // Works

// Create a new wrapper object
Integer myint = 5;

  • To get the object wrapper’s value: intValue(), byteValue(), shortValue(), longValue(), floatValue(), doubleValue(), charValue(), booleanValue().

  • To convert it to a String: .toString().

  • To get the length of the string

    Integer myint = 1000;
System.out.println(myint.toString().length()); // 4, as "1000" has four characters

6. Advanced stuff

Java Exceptions

try

If the block of code fails to be executed then run some failback code.

public class Main {
  public static void main(String[] args) {
    try { // Initial code tried to get executed
      int[] myNumbers = {1, 2, 3};
      System.out.println(myNumbers[10]); // Out of bound error
    } catch (Exception e) { // error in try{} so execute this:
      System.out.println("Something went wrong.");
    } finally { // executed when try-catch is finished
      System.out.println("The 'try catch' is finished.");
    }
  }
}

throw

Use throw to throw a custom error of type ArithmeticException, FileNotFoundException, ArrayIndexOutOfBoundsException, SecurityException.

throw new ArithmeticException("Some error message");
