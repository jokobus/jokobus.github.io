Install Custom Applications on Linux

A guide on installing standalone Linux applications and creating desktop shortcuts (.desktop files) for a seamless desktop experience.

Overview

When using Linux machines where you lack root privileges (sudo), you can still install applications locally. This guide covers how to set up standalone binaries (like VS Code, Obsidian, or custom tools) and integrate them into your desktop environment’s app launcher using .desktop files.

Step-by-Step Installation

1. Download and Prepare

Download the executable or compressed archive (.tar.gz, .zip, .AppImage) for your application. If it’s a folder, move it to a dedicated local bin directory.

Example Path:

/home_local/username/bin/myapp/

but you can of course use another directory as well.

2. Make the application executable

  1. Find the main binary executable file in the folder.

    ls /home_local/username/bin/myapp

  2. Use chmod to grant execution permissions to it

    Let’s say we found the executable to be named myapp as well. If you downloaded a .AppImage, use this file instead.

    chmod +x /home_local/username/bin/myapp/myapp

3. Add to System PATH (Optional)

If you want to run the application from any terminal by just typing its name, you can add it to your .bashrc.

Note: On some systems, the bin folder might already be in your PATH.

  1. Open your bash configuration:
    nano ~/.bashrc
  2. Add this line at the end:
    export PATH="/home_local/username/bin/myapp:$PATH"
  3. Apply changes:
    source ~/.bashrc

4. Create a Desktop Entry

To make the application appear in your app launcher (Super key / Start menu), create a .desktop file.

  1. Create the file:

    nano ~/.local/share/applications/myapp.desktop

  2. Paste the following template and adjust the paths:

    [Desktop Entry]
Version=1.0
Type=Application
Name=My Application
Comment=A short description of the app
Exec=/net/rmc-lx0181/home_local/fuer_jk/bin/myapp/executablefile
Icon=/net/rmc-lx0181/home_local/fuer_jk/bin/myapp/icon.png
Terminal=false
Categories=Utility;Application;

5. Finalize and Refresh

Make the desktop entry itself executable and update the desktop database.

chmod +x ~/.local/share/applications/myapp.desktop
update-desktop-database ~/.local/share/applications

6. Search for your App

Press the Super key (Windows key) and search for your application name.

Troubleshooting Tips

  • Binary not found: Ensure the Exec path is absolute and points to the actual file, not its containing folder.
  • Icon not showing: Ensure the Icon path points to a valid .png or .svg file.
  • Validation: Run desktop-file-validate ~/.local/share/applications/myapp.desktop to check for syntax errors.
